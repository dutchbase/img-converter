**Test 1 — Basic batch flow (REQ-201, REQ-203)**
1. Open http://localhost:3100
2. Drag 3 image files onto the drop zone
3. Verify: All 3 files appear in the queue with "Pending" status badges and original sizes shown
4. Verify: An aggregate count shows "0 / 3 converted"
5. Verify: Each pending row has an X button

**Test 2 — Remove pending item (locked decision)**
1. Click the X on one pending row
2. Verify: That row disappears; queue shows 2 files; aggregate count updates to "0 / 2 converted"

**Test 3 — Shared settings + Convert All (REQ-202, REQ-203, REQ-204)**
1. Drop 3 files again; set format to WebP in the options panel
2. Click Convert All
3. Verify: Drop zone becomes visually disabled (greyed out, cannot drop files)
4. Verify: Convert All button shows a spinner and text like "0/3 converting..."
5. Verify: Each row transitions from "Pending" to "Converting" to "Done" or "Failed"
6. Verify: Done rows show original size, arrow, converted size, and a "Download" link
7. Verify: Aggregate count updates as files complete

**Test 4 — Error resilience + Retry (REQ-207)**
1. Include a file that will fail (e.g., rename a .txt file to .jpg — it fails MIME verification at the API)
2. Click Convert All with a mix of valid and invalid files
3. Verify: The invalid file shows "Failed" status with an error message and Retry button
4. Verify: The valid files complete normally ("Done" status)
5. Click Retry on the failed file
6. Verify: The item shows "Converting" again, then "Failed" again (expected for an invalid file)

**Test 5 — ZIP download (REQ-206)**
1. After all files complete, verify the "Download N files as ZIP" button appears

5. Open the ZIP and verify: contains only successfully converted files with original base names and new extensions (e.g., `photo.webp`)
6. Verify: Individual "Download" links on each done row still work

**Test 6 — Clear queue (locked decision)**
1. Click "Clear queue"
2. Verify: Queue disappears; drop zone returns to its enabled state
3. Verify: Can drop new files immediately

**Test 7 — Drop zone locked during conversion (locked decision)**
