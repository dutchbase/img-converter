  <behavior>
    REQ-101 — HTTP 422 for oversized images:
    - After reading the file into `inputBuffer`, call `sharp(inputBuffer).metadata()` to get dimensions
    - If `(meta.width ?? 0) * (meta.height ?? 0) > 25_000_000`, return `NextResponse.json({ error: "IMAGE_TOO_LARGE", message: "Image dimensions exceed limit" }, { status: 422 })`
    - This check runs BEFORE calling `processImage()` — it is a route-layer guard (processImage also has its own guard from Plan 02, providing defense in depth)

    REQ-102 — Filename sanitization:
    - Replace the current `originalName` derivation with: strip extension, then replace `[^a-zA-Z0-9._-]` with empty string
    - If the result is empty, use `"converted"` as fallback
    - The final `filename` embedded in `Content-Disposition` must contain only safe characters
    - This change is transparent to the user (no client display needed per CONTEXT.md)

    REQ-104 — Magic-byte MIME verification:
    - Install `file-type` package: `npm install file-type`
    - Use dynamic `import()` inside the async handler: `const { fileTypeFromBuffer } = await import('file-type')`
    - Do NOT use a top-level static import — this causes `ERR_REQUIRE_ESM` at startup in Next.js CJS context
    - After converting `file` to `inputBuffer` (Buffer), call `fileTypeFromBuffer(inputBuffer)`
    - If result is undefined OR `detectFormat(detected.mime)` returns null, return HTTP 415 with `{ error: "UNSUPPORTED_FORMAT", message: "File type does not match its contents" }`
    - The existing MIME-based `detectFormat(file.type)` check can remain as a fast pre-filter; the `file-type` check is the authoritative security gate

    ORDER OF CHECKS in route handler (after file size check):
    1. `detectFormat(file.type)` — fast format pre-filter (existing)
    2. Convert to Buffer: `inputBuffer = Buffer.from(await file.arrayBuffer())`
    3. `fileTypeFromBuffer(inputBuffer)` — magic-byte verification (new, REQ-104)
    4. `sharp(inputBuffer).metadata()` — pixel dimension check (new, REQ-101)
    5. Parse remaining form fields, build `ConvertOptions`
    6. Call `processImage(inputBuffer, options)`
    7. Build response with sanitized filename (REQ-102)
  </behavior>
