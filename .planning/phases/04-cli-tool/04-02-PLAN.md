<task type="auto">
  <name>Task 1: Create tsconfig.cli.json</name>
  <files>tsconfig.cli.json</files>
  <action>
Create `tsconfig.cli.json` at the project root with these exact compiler options. The include array MUST be restricted to cli/, lib/, and types/ only — including app/ or components/ causes React JSX compilation failures.

```json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "module": "CommonJS",
    "moduleResolution": "node",
    "outDir": "dist/cli",
    "rootDir": ".",
    "noEmit": false,
    "target": "ES2020",
    "lib": ["ES2020"],
    "paths": {
      "@/*": ["./*"]
    }
  },
  "include": ["cli/**/*.ts", "lib/**/*.ts", "types/**/*.ts"],
  "exclude": ["node_modules", "app", "components", "__tests__", "dist"]
}
```

Key override rationale:
- `"module": "CommonJS"` — required for `require()` in Node.js CLI
- `"moduleResolution": "node"` — matches CommonJS; resolves glob v10 CJS build correctly
- `"noEmit": false` — actually emit compiled JS
- `"outDir": "dist/cli"` — isolate CLI output from Next.js `.next/` artifacts
- `"lib": ["ES2020"]` — drop `dom` and `dom.iterable` which are irrelevant for CLI and cause false type errors
- `"paths"` — re-declared here because `extends` does NOT inherit paths when outDir changes the root
- `"include"` — scoped explicitly to avoid JSX compilation errors from app/ and components/

Note: `cli/index.ts` does not exist yet — that is fine; tsc will still validate the config.
